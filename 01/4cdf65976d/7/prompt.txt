# Fix Dark Zellij Statusline Separators

## Summary
- The remaining gray Powerline separator is not from `--terminal-color-15`.
- Root cause: Zellij leaves SGR bold active on the statusline, and xterm.js defaults `drawBoldTextInBrightColors` to `true`, so ANSI black (`38;5;0`) is rendered as bright black (`#666`) for the `` glyph.
- Fix xterm rendering so bold changes weight only, not ANSI color index.

## Implementation Changes
- In `web/src/features/home/components/organisms/xterm-terminal.tsx`, add `drawBoldTextInBrightColors: false` to the `new XTerm({ ... })` options.
- Keep `minimumContrastRatio: 4.5`; xterm already excludes Powerline/private-use glyphs from contrast adjustment, and normal terminal readability still benefits from it.
- Leave the dark ANSI palette as-is:
  - `--terminal-color-0: #000000`
  - `--terminal-color-15: #ffffff`
  - `--terminal-color-8: #666666`

## Test Plan

  - terminal statusline still renders white segment backgrounds,
  - black-on-white `` separators render black, not gray/bright-black.
