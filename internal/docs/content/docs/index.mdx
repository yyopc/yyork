yyork docs are the project decision log for the local agent workspace.

Start with [Design decisions](/docs/decisions) when you need the current
product and architecture rationale. Use [Architecture](/docs/architecture) for
deeper implementation notes, and [Investigations](/docs/investigations) for
time-bound debugging records.

## Current map

| Section | Use it for |
| --- | --- |
| [Design decisions](/docs/decisions) | Durable choices that should guide future implementation. |
| [Architecture](/docs/architecture) | How specific pieces work today. |
| [Investigations](/docs/investigations) | Past incidents and root-cause notes. |

## Writing rule

These docs should grow with yyork. When a feature lands because of a product or
architecture choice, add the decision here and link to the PRD, code owner, or
investigation that made the choice concrete.
